用户要求：只要不是非常简单的代码，每行逻辑都要加注释。在代码中明确选择「每行都注释」密度--每个语句 / 逻辑块都加行级注释，只跳过空行 / 括号 / 纯声明。

**Why:** 用户在重构时，先要求精简注释，随后改口要求每行逻辑都注释。说明该用户 / 项目偏好高密度行级注释，便于阅读维护，而非追求注释最少化。

**How to apply:** 写或改非平凡代码时，为每个语句补充行级 `//` 注释说明该行作用。注释要说清「这行做什么」，而非仅「为什么」。明显的单行仍可省，但条件判断、赋值、调用、返回值等都要注释。除非用户要求，不主动精简已有注释。注释要详，结构要简。
